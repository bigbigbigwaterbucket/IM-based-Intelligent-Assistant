# PPT/演示稿生成 Skill

## 通用规则

- 幻灯片必须面向听众，不描述 agent 自身的思考、规划、工具调用或中间状态。
- 不要出现“意图分析”“执行计划”“工具调用”等内部过程。
- 每页只保留 2 到 5 个重点，避免长段落。
- 如果材料来自聊天上下文，按背景、关键讨论、结论、待办、风险组织。
- 如果缺少真实上下文，标注“待补充信息”，不要编造事实。
- 只生成可被 genppt 转换的 Markdown，不生成 Slidev frontmatter、HTML、演讲稿或备注。

## Markdown 转 PPTX 规则

- `# 标题`：创建一张新幻灯片。每一页都必须以一级标题开始。
- `## 子标题`：作为当前幻灯片的标题或内容小标题。
- `- 列表`、`* 列表`、`+ 列表`：转换为无序列表。
- `1. 列表`：转换为有序列表。
- 三反引号代码块：转换为代码块。
- `---`：分隔符，可用于结束当前页，但推荐直接用下一个 `# 标题` 新建下一页。
- 普通段落会作为正文文本；避免长段落，优先拆成列表。
- 不要使用 `<!-- speaker notes -->`、`::`、Vue 组件、布局 frontmatter 或其他 Slidev 专属语法。

## 默认结构

1. 标题页：主题、日期或汇报对象。
2. 背景：为什么讨论这个问题。
3. 关键内容：最重要的讨论点或方案。
4. 结论：已经明确的决定。
5. 下一步：行动项、负责人、时间。
6. 风险与待确认：仍需补充的信息。
